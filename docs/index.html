Who it's for

macOS & Linux developers using Claude Code who run more than one agent at a time — and want fleet-level awareness without babysitting each tab. If you're terminal-first, keyboard-first, and want your tooling open and self-hosted, this is built for you.

Fleet-level awareness

Every session is a real claude subprocess. Everything is measured, not simulated.

$

Live cost & context

Per-agent token, cost, and context-window tracking straight from Claude's own usage — with weekly spend and budget caps.

⋔

Sub-agent fan-out

See when an agent spawns parallel sub-agents, with fan-out count and folded token accounting. No other tool shows this.

▸

Triage at a glance

Each card answers "does this need me, when, and what next" — todo burndown plus a status-driven next-action verb.

↵

Zoom into any session

Drop into a full PTY view of any agent — scroll history, toggle stats and tools panels, send input — then pop back to the grid.

✓

Approval routing

Permission prompts surface on the card and route to a keystroke. Cycle plan / auto / acceptEdits modes without leaving the fleet.

◈

Themes & restore

Seven palettes including green-phosphor Matrix. Save & restore sessions across restarts. Yours to re-theme.

Why not tmux or cmux?

Not a generic multiplexer, not a native app — a TUI that understands Claude Code agents. Runs alongside tmux and over SSH.

Get started

Requires Node 20+, the claude CLI on your PATH, and a 24-bit-color terminal.

# Run it now, no install
npx @bluearch/mission-control

# …or install the mc command globally
npm install -g @bluearch/mission-control
mc
# From source (contributors)
git clone https://github.com/xxyjoel/ba-mission-control.git
cd ba-mission-control
npm install && npm start
